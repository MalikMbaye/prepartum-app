# PROMPT MB-1: DATABASE SCHEMA
## Paste everything between the triple backticks into Replit Agent chat

```
Update the PrePartum database schema for the full Memory Bank feature.

## 1. Drop and recreate the memories table with the full schema:

CREATE TABLE IF NOT EXISTS memories (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  
  -- Core fields
  type TEXT NOT NULL CHECK (type IN ('text', 'photo', 'voice', 'video')),
  title TEXT,
  content TEXT,
  memory_date DATE NOT NULL DEFAULT CURRENT_DATE,
  
  -- Media
  media_urls TEXT[],         -- array for multiple photos or single file URL
  media_thumbnail_url TEXT,  -- first photo or auto-generated thumb
  
  -- Organization
  tags TEXT[],
  milestone_id UUID REFERENCES milestones(id) ON DELETE SET NULL,
  trimester INTEGER,         -- 1, 2, or 3 — calculated on insert
  
  -- Metadata
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_memories_user_id ON memories(user_id);
CREATE INDEX IF NOT EXISTS idx_memories_memory_date ON memories(memory_date);
CREATE INDEX IF NOT EXISTS idx_memories_user_date ON memories(user_id, memory_date DESC);

## 2. Create a Supabase storage bucket for memory media:

Run this in the Supabase dashboard SQL editor (Storage API):
- Bucket name: memory-media
- Public: false (private, user-authenticated access)
- File size limit: 50MB
- Allowed MIME types: image/jpeg, image/png, image/webp, audio/mpeg, audio/mp4, audio/wav, audio/m4a, video/mp4, video/quicktime

## 3. Create an RLS policy for the memories table:

ALTER TABLE memories ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only see their own memories"
  ON memories FOR ALL
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert their own memories"
  ON memories FOR INSERT
  WITH CHECK (auth.uid() = user_id);

Run all SQL, confirm the table and indexes exist, then tell me it's done.
```
