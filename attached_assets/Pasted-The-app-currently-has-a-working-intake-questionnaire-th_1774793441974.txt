The app currently has a working intake questionnaire that saves to profile.profileFlags.persona. We need to wire that persona value into every layer of the app so each user gets a genuinely different experience.

The 5 personas are:
- anxious_planner
- supported_nurturer
- solo_warrior
- healing_mother
- faith_anchored

PART 1 — PERSONA UTILITY LAYER

Create a new file: lib/persona.ts

This file is the single source of truth for all persona-driven content. Export the following:

1. A PERSONA_CONFIG object with an entry for each persona containing:
   - displayName: string
   - partnerTerm: string  
     → supported_nurturer: "your partner"
     → solo_warrior: "your support person"
     → healing_mother: "your support person"
     → anxious_planner: "your partner or support person"
     → faith_anchored: "your partner or support person"
   - villageTerm: string
     → solo_warrior: "your village"
     → all others: "your people"
   - homeScreenEmphasis: 'tasks' | 'milestones' | 'strength' | 'body' | 'surrender'
     → anxious_planner: 'tasks'
     → supported_nurturer: 'milestones'
     → solo_warrior: 'strength'
     → healing_mother: 'body'
     → faith_anchored: 'surrender'
   - affirmationTone: 'grounding' | 'celebratory' | 'empowering' | 'gentle' | 'spiritual'
   - promptPace: 'structured' | 'open' | 'direct' | 'slow' | 'reflective'
   - weeklyFocusLabel: string (a short phrase shown on the home screen)
     → anxious_planner: "Preparing with intention"
     → supported_nurturer: "Growing together"
     → solo_warrior: "Finding your strength"
     → healing_mother: "Moving gently forward"
     → faith_anchored: "Trusting the process"
   - taskBoardLabel: string
     → anxious_planner: "Your preparation checklist"
     → supported_nurturer: "Things to do together"
     → solo_warrior: "What you can do today"
     → healing_mother: "At your own pace"
     → faith_anchored: "Preparing with purpose"

2. A getPersonaConfig(persona: string) function that returns the config for that persona, falling back to supported_nurturer as default.

3. A sanitizeForPersona(text: string, persona: string) function that:
   - Replaces "your partner" with the persona's partnerTerm
   - Replaces "your village" with the persona's villageTerm
   - Replaces "partner or support person" with the persona's partnerTerm
   - Returns the sanitized string

PART 2 — DAILY PROMPT DELIVERY

Update the endpoint GET /api/users/:userId/daily-prompts (or wherever daily prompts are fetched) to filter by persona.

The updated query logic:
1. Get the user's persona from their profile (profile.profileFlags.persona)
2. Query prompts WHERE:
   - week_number = currentWeek (calculated from due date)
   - persona_tags = 'All' OR persona_tags CONTAINS userPersona
   - ordered by day ASC
3. Return 3 prompts per day (one Mindset, one Relationships, one Physical) when possible — prioritize persona-tagged prompts over 'All' prompts when both exist for the same category

On the frontend, wherever prompt text is displayed, pass it through sanitizeForPersona(promptText, userPersona) before rendering.
